    HTTP 协议基础知识
    2018-11-10 · 吾儿滨滨

    什么是 HTTP ？

    HTTP 协议是 Hyper Text Transfer Protocol (超文本传输协议) 的缩写，用于从万维网服务器传输超文本到本地浏览器的传输协议。

    设计 HTTP 最初的目的设计为了提供一种发布和接收 HTML 页面的方法，现在 HTTP 是万维网数据通信的基础。

    这是我们通过浏览器和服务端对话的方式，大到网页的刷新，小到按钮的点击，都是通过http协议来实现的。

    HTTP 工作原理

    HTTP 协议工作在客户端-服务端架构上，即浏览器作为客户端通过 URL 向 Web 服务器发送所有请求，Web服务器接收请求后，向浏览器发送响应信息。

    HTTP 的特点？

    1. HTTP 是无连接的

      即每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。这种方式可以节省传输时间。

    2. HTTP 是媒体独立的

      只要客户端和服务器知道如何处理的数据内容，HTTP 允许传输任意类型的数据对象。传输的类型由 Content-Type加以标记。

    3. HTTP 是无状态的的

      无状态是指协议对于事务处理没有记忆，后续处理需要前面的信息，则必须重传。

    http 和 https 的区别？

    HTTPS 是 HTTP 协议的安全版本，HTTP 协议的数据传输是明文的，是不安全的。HTTPS 使用了 SSL/TLS 协议进行了加密处理。

    Chrome 68 浏览器已经把所有未采用 HTTPS 的网站都标记为 「不安全」 网站。

    HTTP 的请求方法

    HTTP 有多种请求方法，绝大部分情况下，我们只需了解 GET 和 POST。

    GET 和 POST 都是表单提交数据的两种基本方式，GET 请求数据通过域名后缀 url 传送，用户可见，不安全。POST 请求数据通过在请求报文正文里传输，相对比较安全。

    关于 GET 和 POST 的区别，这篇文章讲的很有趣！

    HTTP 状态码

    当我们访问网站时，有时候，网站会出现「404 错误」的字样，我们知道这就是页面丢失了。

    作为一种标准的 HTTP 返回代码，404 被用来表示请求的资源不存在。

    常见的 HTTP 状态码还有 200、301、500 等。

    HTTP 状态码根据第一个数字不同，可以分为 5 钟类型：

    分类 描述
    1** 信息，服务器收到请求，需要请求者继续执行操作
    2** 成功，操作被成功接收并处理
    3** 重定向，需要进一步的操作以完成请求
    4** 客户端错误，请求包含语法错误或无法完成请求
    5** 服务器错误，服务器在处理请求的过程中发生了错误

    常见 HTTP 状态码：

    • 200 (OK)：请求成功，一般用于 GET 和 POST 请求。
    • 301 (Moved Permanently)：永久移动，请求的网页已永久移动到新位置。 服务器返回此响应（对 GET 或 HEAD 请求的响应）时，会自动将请求者转到新位置。
    • 404 (Not Found)：未找到，服务器无法根据客户端的请求找到资源（网页）。通过此代码，网站设计人员可设置 “您所请求的资源无法找到” 的个性页面。
    • 500 (Internal Server Error)：服务器内部错误，无法完成请求。
