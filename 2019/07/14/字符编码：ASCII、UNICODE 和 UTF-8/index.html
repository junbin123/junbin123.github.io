    字符编码：ASCII、UNICODE 和 UTF-8
    2019-07-14 · 吾儿滨滨

    1. ASCII 码

      • 计算机内部，所有信息最终都是一个二进制值。
      • 每个二进制位（bit）有0和1两种状态。
      • 8 个二进制位可组合出 256 种状态，称为一个字节（byte），每一个状态对应一个符号，即00000000到11111111。
      • ASCII 码一共规定了 128 个字符编码，这 128 个符号只占用一个字节后面 7 位，最前面同意规定为0。

    2. 非 ASCII 编码

      • 英语 128 个字符编码就够了，但用来表示其他语言，却不够。
      • 欧洲其他国家，如法国，利用字节第一个二进制位来编入新的符号。
      • 不同国家，同一个 ASCII 表示不同字符，造成混乱。
      • 汉字有多达 10 万个左右，需要引入新的编码方式。

    3. Unicode

      • Unicode 是个很大的集合，可容纳 100 多万个字符，保证每个字符编码都不一样
      • Unicode 只是一个符号集合，规定符号二进制代码，却没有规定这个二进制代码应该如何储存。
      • 有些字符

    4. UTF-8

      • UTF-8 是 Unicode 的实现方式之一，此外还有UTF-16、UTF-32，这些在互联网基本不用。

      • UTF-8 最大的一个特点，就是它是一种可变的编码方式。它可以使用 1~4 个字节表示一个符号，根据不同的符号而变化字节长度。

      • UTF-8 编码规则很简单：如果一个字节的第一位是0，则这个字节单独就是一个字符（同 ASCII 码）；如果第一位是1，则连续有多少个1，就表示当前字符占用多少个字节。（如图：x 表示可编码的位置）
