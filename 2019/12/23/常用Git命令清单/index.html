    常用 Git 命令清单
    2019-12-23 · 吾儿滨滨

    Git 是每个程序员必备的技能，熟练掌握 Git 命令，能极大提高工作效率。

    下面是我整理的常用 Git 命令速查清单。

    1. 新建代码库

    # 在当前目录初始化 Git 代码库
git init
# 新建一个文件夹，并初始化 Git 代码库
git init project
# 下载一个项目
git clone https://github.com/junbin123/likeday.git

    2. 查看代码库

    # 显示有变更的文件
git status

# 显示当前分支的 commit
git log
git log --oneline

# 显示统计信息
git log --stat

# 根据提交内容查找对应 commit
git log -grep="Set heading"
git log --grep="Set heading" --stat

# 显示每个 commit 的tag、分支
git log --decorate
git log --oneline --decorate

# 查看详细修改
git log -p
git show

# 查看详细修改，只看最近两条
git log -p -2

# 查看详细修改，忽略空行
git log -p -w

# 查看统计信和详细修改
git log -p --stat

# 显示所有分支的 commit 情况
git log --oneline --graph --all

    3. 暂存区操作

    # 将文件index.html移到暂存区
git add index.html

# 将所有更改文件移到暂存区
git add .

# 从暂存区和工作区删除index.html
git rm index.html

# 停止追踪 index.html 文件，但会保存在工作区
git rm -cached index.html

    4. 代码提交

    # 将修改提交到仓库，修改信息为Initial commit
git commit -m "Initial commit"

# 使用一次新的 commit，替代上一次提交
# 如果代码没有任何新变化，则用来改写上一次commit的提交信息
git commit --amend -m "init commit"

    5. 分支操作

    # 列出所有本地分支
git branch

# 列出所有远程分支
git branch -r

# 列出所有本地分支和远程分支
git branch -a

# 新建名为 sidebar 的分支
git branch sidebar

# 新建名为 sidebar 的分支，并切换
git branch -b sidebar

# 从 commit 为 ed5f0dd 上新建一个分支 heading
git branch heading ed5f0dd

# 删除 sidebar 分支（需切换到其他分支） 
git branch -d sidebar

# 切换到 sidebar 分支（删除原分支中的 commit 引用的所有文件，替换为 sidebar）
git checkout sidebar

# 合并 sidebar 分支到当前分支
git merge sidebar

# 删除远程分支 sidebar
git push origin --delete sidebar
git branch -dr sidebar

    6. 标签

    # 列出所有 tag
git tag

# 在当前 commit 添加一个 tag
git tag v1.0.0

# 在指定 commit 上添加一个 tag
git tag v1.0.0 ed5f0dd

#为当前 commit 添加 v1.0.0 的tag（带注释：标签创建者、标签创建日期、标签信息）
git tag -a v1.0.0

# 删除本地 tag v1.0.0
git tag -d v1.0.0

# 提交所有 tag
git push origin --tags

# 提交指定 tag
git push origin v1.0.0

    7. 远程同步

    # 显示所有远程仓库
git remote -v

# 显示远程仓库 origin 的信息
git remote show origin

# 增加一个新的远程仓库 staging
git remote add staging https://github.com/junbin123/likeday.git

# 上传 master 分支到远程仓库 origin
$ git push origin master

# 强行推送当前分支到远程仓库 origin，即使有冲突
$ git push origin --force

# 推送所有分支到远程仓库 origin
$ git push origin --all
