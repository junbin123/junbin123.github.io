    Git 常用命令
    2020-05-02 · 吾儿滨滨

    Git 易学难精。

    Git 只在乎文件内容，不在乎文件名称。

    Git 分支廉价好用。

    常用命令

    # 查看信息
git config user.name
git config user.email

# 修改信息
git config --global user.name "junbin123"
git config --global user.email "540965117@qq.com"

    基本操作

    # 显示一行显示所有commit
git log --oneline

# 显示统计信息
git log --stat

# 查看详细修改
git log -p
git show

# 从暂存区和工作区删除index.html
git rm index.html

# 从暂存区删除index.html
git rm -cached index.html

# 修改最近一次 commit 的 msg，同时把暂存区的修改提交到仓库
git commit --amend -m "feature header"

# 重做上一次commit，并包括指定文件的新变化
git commit --amend [file1] [file2] ...

# 显示你已经保存但是尚未 commit 的更改
git diff

    标签

    # 为当前commit添加v1.0的tag
git tag v1.0.0

#为当前commit添加v1.0的tag，带注释（标签创建者、标签创建日期、标签信息）
git tag -a v1.0.0

# 删除名称我v1.0的tag
git tag -d v1.0

# 删除SHA为a87984的tag
git tag -a v1.0 a87984

    分支

    # 创建并切换到 footer 分支，同时分支的起点与 master 分支的一样
git checkout -b footer master

# 从 commit 为 ed5f0dd 上创建分支 sidebar
git branch sidebar ed5f0dd

# 删除本地 sidebar 分支（此时位于其他分支） 
git branch -d sidebar

# 删除远程 sidebar 分支
git push origin -d sidebar

# 合并sidebar分支（此时位于其他分支）
git merge sidebar

    撤销

    git revert 用一次新的 commit 回滚之前的 commit，git reset 直接删除指定的 commit。 

    # 撤销前一次 commit
git revert HEAD

# 撤销前前一次 commit
git revert HEAD^

# 撤销 commit 为 fa042c 的操作
git revert fa042c

# 撤回你最近一次的commit，把修改保存在暂存区
git reset --soft HEAD^

# 撤回你最近一次的commit，把修改也删除掉
git reset --hard HEAD^
    • --hard：清除 commit，回收站；
    • --soft：将 commit 的更改移至暂存区；
    • --mixed：取消暂存已被 commit 的更改，工作区。

    其他

    .gitignore

    希望 git 忽略一些文件，可以在 .gitignore 文件，使用通配符配置。

    # 忽略images文件夹中jpg格式的图片
samples/*.jpg
#
    • 空白行作为空格
    • # - 将行标记为注释
    • * - 与 0 个或多个字符匹配
    • ? - 与 1 个字符匹配
    • [abc] - 与 a、b 或 c 匹配

    Vim编辑

    • 退出：:q
    • 保存并退出v：ese→:wq

    一些场景

    • 设置本地 dev 分支跟踪远程 dev 分支，以后只需执行 git pull 就行了。
    git branch --set-upstream-to=origin/dev dev
    • 有时候同一个分支，远程的和本地的都被修改的面目全非了，如果想要把本地的替换成远程的，用下面的命令
    git fetch --all
# master为要修改的分支名
git reset --hard origin/master
git pull

    • -u 指定 origin 为默认主机，后面可以不加任何参数，直接使用 git push 了。

      不带任何参数的 git push，默认只推送当前分支，这叫做 simple 方式。此外，还有一种 matching 方式，会推送所有有对应的远程分支的本地分支。Git 2.0 版本之前，默认采用 matching 方法，现在改为默认采用 simple 方式。

    git push -u origin master
    • 拉取远程分支 dev 到本地
    git checkout -b dev oriign/dev
