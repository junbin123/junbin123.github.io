    理解 BFC
    2020-02-08 · 吾儿滨滨

    BFC（Block Formatting Context，块级格式化上下文） 是 CSS 布局中的一个概念，是元素的一种特性。

    它是一个独立的渲染区域，容器里面的子元素不会影响到外面元素。

    通俗的讲，可以把 BFC 理解成一个封闭的大箱子，不管里面的元素如何变化，都不会影响到外部。

    为了更好地理解，先从一个例子说起。

    HTML

    <div id="father">
  <div id="child1">I am a floated box!</div>
  <div id="child2">I am content inside the container.</div>
</div>

    CSS

    #father {
  background: lightblue;
}

#child1 {
  width: 100px;
  height: 100px;
  background: lightgreen;
  float: left;
}

    上面的例子中，父元素下有两个子元素，其中子元素 child1 设置浮动，因此脱离了文档流。

    父元素仅仅包含了子元素 child2。

    现在，创建一个包含这个浮动元素的 BFC，通常的做法是设置父元素overflow: auto。

    CSS

    #father {
  background: lightblue;
  overflow: auto; // 创建BFC
}

#child1 {
  width: 100px;
  height: 100px;
  background: lightgreen;
  float: left;
}

    overflow 属性告诉浏览器，要怎样处理溢出的内容。

    如何创建 BFC？

    可以通过为元素添加属性，触发 BFC 特性。其中，根元素<body>本身就是具有 BFC 特性。

    元素只要满足下面任意一条，即可触发 BFC 特性。

    • overflow: auto/hidden/scroll
    • display: inline-block/table-cells/flex/grid/flow-root
    • 浮动元素：float: left/right
    • 绝对定位元素：position: absolute/fixed

    BFC 的作用

    关于 BFC 的作用可以罗列三四条，实际上，主要的只有两个，其他的都是在此基础的扩展。

    分别为：

    1. 避免 margin 塌陷。

    2. 处理浮动元素。

      包含浮动元素 & 避免元素被浮动元素覆盖。

    1. 避免 margin 塌陷

    margin 塌陷的意思是，垂直方向上的两个盒子，外边距会重叠在一起（取 margin 最大的那个）。

    下面通过一个例子来说明。

    HTML

    <div id="brother1"></div>
<div id="brother2"></div>

    CSS

    #brother1 {
  width: 200px;
  height: 60px;
  margin: 30px;
  background: lightgreen;
}

#brother2 {
  width: 200px;
  height: 60px;
  margin: 30px;
  background: lightblue;
}

    例子中，两个 div 元素均设置 margin 为 30px，正常情况下，它们之间的间隔应该为 60px。

    实际上，它们的间隔为 30px。

    可以理解为这两个 div 元素处于同一个 BFC 容器。

    首先，这是 CSS 的一种规范，若要避免 margin 塌陷，可以将它们放在不同的 BFC 容器中。

    代码修改如下。

    HTML

    <div class="container">
  <div id="brother1"></div>
</div>
<div class="container">
  <div id="brother2"></div>
</div>

    CSS

    .container {
  overflow: hidden;
}

#brother1 {
  width: 200px;
  height: 60px;
  background: lightgreen;
  margin: 30px;
}

#brother2 {
  width: 200px;
  height: 60px;
  margin: 30px;
  background: lightblue;
}

    2. 处理浮动元素

    关于 BFC 特性如何包含浮动元素，第一个例子已经说过。

    下面通过一个例子，理解 BFC 特性如何避免元素被浮动元素覆盖。

    HTML

    <div id="brother1">我是浮动元素</div>
<div id="brother2">
    这雨愈下愈老成，水点贯串作丝，河面上像出了痘，无数麻瘢似的水涡，随生随灭，息息不停，到雨线更密，又仿佛光滑的水面上在长毛。
</div>

    CSS

    #brother1 {
  width: 100px;
  height: 100px;
  background: lightgreen;
  float: left;
}

#brother2 {
  width: 200px;
  height: 200px;
  background: lightblue;
}

    上面的例子中，相邻的两个元素，第一个元素设置浮动，导致第二个元素的内容出现文字环绕的效果。

    通过触发第二个元素的 BFC 特性，为元素添加overflow: hidden，就会变成：

    这个方法也可以用来实现两列自适应布局。

    左边宽度固定，右边宽度自适应。

    进阶

    Box 和 FC

    BFC 是 FC 的一种方案，相对应的，还有 IFC（Inline formatting context，行内格式化上下文）、GFC、FFC。

    其中 CSS2.1 只有 BFC 和 IFC，CSS3 新增了 GFC 和 FFC。

    理解 FC 之前，先了解一下 Box。

    Box 是 CSS 布局中最基本的单位。直观的说，就是一个页面有很多个 Box 组成。

    常用的 Box 类型有两个，分别是 block box 和 inline box，由元素的display属性决定。不同的 box 对应不同渲染方式。

    下面，再来了解一下什么是 FC。

    FC（Formatting Context，格式化上下文）是 CSS2.1 规范中的一个概念，它是页面中的一块渲染区域，并且有一套渲染规则。

    这里概念比较多，可以用下面这张表总结一下。

    box 类型 对应元素 特性 渲染方式 display 值
    block box 块级元素 BFC block-level block/list-item/table
    inline box 行内元素 IFC inline-level inline/inline-block/inline-table

    定位方案

    很多 Box 放在一起，需要一套机制规定它们的位置。

    于是就有了定位方案，浏览器会根据元素的类型和上下文对这些元素进行定位。

    有三种方案，分别是：常规流（Normal Flow）、浮动（Float）、绝对定位（Absolute Position）。
